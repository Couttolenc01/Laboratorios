Preguntas

¿Cuál es la diferencia entre Internet y la World Wide Web?


Internet es la red global de computadoras que permite la comunicación y el intercambio de información, mientras que la World Wide Web es un servicio que se ejecuta en la parte superior de Internet que permite a los usuarios acceder y compartir información.



¿Cuáles son las partes de un URL?


Protocolo, Nombre de dominio, Ruta del recurso, Parámetros, Ancla (protocolo://nombre_de_dominio/ruta_del_recurso?parámetros#ancla)



¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?


Cada uno de los métodos HTTP tiene un propósito específico en la comunicación entre el cliente y el servidor. GET se utiliza para obtener información, POST y PUT para enviar información, PATCH para actualizar parcialmente un recurso y DELETE para eliminar un recurso. HEAD se utiliza para obtener información sobre un recurso sin descargar todo su contenido.




¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?


Se debe utilizar el método POST al enviar un formulario HTML que incluya información sensible como los datos de inicio de sesión, ya que proporciona seguridad y privacidad al usuario.




¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?


El método HTTP utilizado cuando se accede a una página a través de un URL en un navegador web es el método HTTP GET.




Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?


Una respuesta HTTP con código 200 indica que la solicitud se procesó correctamente y no ocurrió ningún error.




¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?


Es responsabilidad del desarrollador corregir un error 404 reportado por un usuario para garantizar la funcionalidad y la accesibilidad del sitio web.




¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?


Es responsabilidad del desarrollador corregir un error 500 reportado por un usuario para garantizar la funcionalidad y la accesibilidad del sitio web.




¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.


Cuando un atributo HTML5 está "depreciado" o "desaprobado" (en inglés "deprecated"), significa que se considera obsoleto o en desuso, y que puede ser eliminado en versiones futuras de HTML. Esto no significa que el atributo ya no funcione, pero se recomienda no usarlo ya que puede haber alternativas más adecuadas y modernas.
Basefont: Se recomienda usar CSS para definir la fuente base del documento en lugar de este elemento, Big: Se recomienda usar CSS para definir la fuente de tamaño grande en lugar de este elemento.






¿Cuáles son las diferencias principales entre HTML 4 y HTML5?


HTML5 ha mejorado significativamente la funcionalidad y la flexibilidad de la web, con nuevas características que mejoran la experiencia del usuario, la accesibilidad y la compatibilidad con dispositivos móviles, además de proporcionar una estructura semántica clara para el contenido web.




¿Qué componentes de estructura y estilo tiene una tabla?


Componentes estructurales: Encabezado de la tabla, Cuerpo de la tabla, Filas, Columnas y Celdas. Mientras que los componentes de estilo son el Borde, Fondo, Fuente, Alineación y Resaltado.




¿Cuáles son los principales controles de una forma HTML5?


Algunos de los principales controles de formularios en HTML5 son: Texto, Área de texto, Checkbox, Radio, Selección, Botón, Fecha, Hora, Número y Archivo.

¿Cuál es el ciclo de vida de los sistemas de información?


El ciclo de vida de los sistemas de información es un modelo que describe las diferentes etapas por las que pasa un sistema de información, desde la identificación de la necesidad de información hasta su implementación y mantenimiento.




¿Cuál es el ciclo de desarrollo de sistemas de información?


El ciclo de desarrollo de sistemas de información (SDLC, por sus siglas en inglés) es un proceso secuencial y estructurado que se utiliza para desarrollar y mantener sistemas de información. El SDLC consta de varias etapas que describen los pasos necesarios para planificar, diseñar, implementar y mantener un sistema de información.
