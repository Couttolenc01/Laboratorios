Seccion de preguntas

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Usar JavaScript para verificar la validez de los inputs de un formulario antes de enviar los datos al servidor puede mejorar la experiencia del usuario, ahorrar tiempo y recursos del servidor, y aumentar la seguridad del formulario.


¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Puedes deshabilitar javascript directamente del navegador o si no te puedes ir directamente a la página que es de interes o hacer directamente el form.submit u otra cosa colocando codigo javascript en la url y con eso se salta la validación .


Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

JavaScript se utiliza comúnmente para validar los datos que se ingresan en formularios o entradas de usuario en aplicaciones web, ya que puede proporcionar una validación rápida y eficiente del lado del cliente, sin necesidad de enviar los datos al servidor.
La validación del lado del cliente es importante porque puede proporcionar una mejor experiencia de usuario al informar al usuario sobre cualquier error en tiempo real, en lugar de esperar a que se envíen los datos al servidor y recibir un error. el código JavaScript que realiza la validación se ejecuta en el navegador del usuario y, por lo tanto, puede ser modificado o desactivado por un atacante malintencionado. Es por eso que también se necesita validación en el servidor para garantizar que los datos enviados sean válidos y seguros.
En resumen, la validación de JavaScript es útil para proporcionar una experiencia de usuario mejorada al informar al usuario sobre los errores de entrada en tiempo real, pero no es suficiente para garantizar la seguridad de los datos, y se necesita validación adicional en el lado del servidor para garantizar la seguridad de los datos.
